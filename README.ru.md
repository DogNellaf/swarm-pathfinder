# PSO Agent Navigation

> [English](README.md) | [Русский](README.ru.md)

Система **оптимизации роем частиц (PSO)**, которая подбирает оптимальные параметры навигации для жадного агента, перемещающегося по сетке с типизированной местностью.

---

## Обзор

Агент движется от стартовой ячейки к целевой на квадратной сетке.  Каждая ячейка принадлежит одному из *N* типов местности, а агент выбирает следующий шаг жадным образом, используя функцию перехода

```
F(r, t, α, β) = α[t] + β / r
```

где

| Символ | Значение |
|--------|----------|
| `r` | Манхэттенское расстояние от кандидата до цели |
| `t` | Тип местности **текущей** ячейки |
| `α[t]` | Вес для типа местности `t` |
| `β` | Вес близости к цели |

В каждой ячейке агент переходит в непосещённого соседа с максимальным `F`.  Целевая функция `W` — сумма значений `F` вдоль маршрута.  PSO минимизирует `W`, подбирая вектор `α` и скаляр `β`.

## Установка

Требуется Python 3.9+.  Сторонние зависимости для ядра алгоритма не нужны.  Для запуска тестов необходим `pytest`.

```bash
pip install pytest
```

---

## Использование

### Запуск оптимизатора

```bash
python main.py
```

Скрипт загружает данные из `data/`, запускает PSO и выводит наилучшее `W`, стандартное отклонение роя, оптимизированные `α` и `β`, а также итоговый маршрут.

### Генерация новых случайных данных

```bash
python input_generator.py
```

### Расчёт W для сохранённых параметров без оптимизации

```bash
python w_solution.py
```

### Прямой запуск PSO

```bash
python swarm.py
```

## Настройка

Все настраиваемые константы находятся в [settings.py](settings.py):

| Константа | По умолчанию | Описание |
|-----------|-------------|----------|
| `MAP_SIZE` | `10` | Длина стороны квадратной сетки |
| `LANDSCAPES_COUNT` | `10` | Число типов местности |
| `SWARM_SIZE` | `200` | Число частиц PSO |
| `STEPS_COUNT` | `1000` | Число итераций PSO |
| `CURRENT_VELOCITY_RATIO` | `0.1` | Вес инерции `w` |
| `LOCAL_VELOCITY_RATIO` | `1.0` | Когнитивный коэффициент `c1` |
| `GLOBAL_VELOCITY_RATIO` | `5.0` | Социальный коэффициент `c2` |
| `ALPHA_MIN / ALPHA_MAX` | `0 / 1` | Границы поиска для каждого `α` |
| `BETA_MIN / BETA_MAX` | `0 / 1` | Границы поиска для `β` |
| `WILL_BACKUP` | `False` | Сохранять `.backup`-копию при перегенерации данных |

## Описание алгоритма

### Обновление скорости PSO (стандартная форма)

```
v(t+1) = w · v(t)
        + c1 · r1 · (pbest − x(t))
        + c2 · r2 · (gbest_informant − x(t))
```

Каждая частица выбирает случайное *окружение* из `[neighbor_min, neighbor_max]` частиц, находит в нём информатора с наименьшим личным `W` и обновляет скорость с учётом его позиции.  После обновления скорости частица вычисляет значение целевой функции в новой точке, что обеспечивает обучение на каждой итерации.

Пространство поиска имеет размерность `LANDSCAPES_COUNT + 1`: по одному значению `α` на тип местности и одно значение `β`.  Все координаты ограничиваются отрезком `[0, 1]` после каждого обновления скорости.
